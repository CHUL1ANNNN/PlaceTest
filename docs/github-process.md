# GitHub-процесс для Codex (Task 12)

Цель: стандартизировать запуск задач, чтобы Codex стабильно работал через Issue → branch → PR → review → merge.

## 1) Каждая задача = отдельный GitHub Issue

Для каждой задачи (Task 1…Task 12 и далее) создаётся отдельный Issue.

Шаблон содержания Issue:

- **Цель**: зачем делаем задачу, какой бизнес/технический результат нужен.
- **Вход**: какие данные/артефакты получает модуль или процесс.
- **Выход**: какой ожидается результат (структура, статус, ссылки, файлы и т.д.).
- **Критерии готовности**: проверяемые условия приёмки.

Рекомендуемый заголовок: `Task N: <краткое название>`.

## 2) Codex работает в отдельной ветке

Для каждого Issue создаётся отдельная рабочая ветка.

Рекомендуемый формат ветки:

- `task/<issue-number>-<short-slug>`

Примеры:

- `task/17-photo-import`
- `task/18-ai-client-validation`

## 3) Рабочий цикл: branch → PR → review → merge

1. Создать ветку от актуальной основной ветки.
2. Реализовать изменения только в рамках текущего Issue.
3. Сделать атомарные коммиты с понятными сообщениями.
4. Открыть PR с привязкой к Issue.
5. Пройти ревью, внести правки по комментариям.
6. После одобрения — merge в основную ветку.

## 4) Что обязательно указывать в PR

- **Motivation / Зачем** — кратко, зачем изменение нужно.
- **Description / Что сделано** — список основных изменений по файлам/модулям.
- **Testing / Проверки** — какие команды запускались и результат.
- Ссылку на Issue в формате: `Closes #<issue_number>`.

## 5) Definition of Done для Codex-задачи

Задача считается завершённой, если:

- есть Issue с заполненными: цель/вход/выход/критерии готовности;
- изменения сделаны в отдельной ветке;
- открыт PR с корректным описанием и ссылкой на Issue;
- ревью пройдено и комментарии закрыты;
- PR влит в основную ветку.

## 6) Практический шаблон Issue

```md
## Цель
<описание цели>

## Вход
- ...

## Выход
- ...

## Критерии готовности
- [ ] ...
- [ ] ...
```

## 7) Практический шаблон PR

```md
## Motivation
- ...

## Description
- ...

## Testing
- ✅ <command>
- ⚠️ <command> (причина)

Closes #<issue_number>
```

---

Этот процесс обязателен для всех новых задач, чтобы Codex работал как предсказуемый агент разработки: через коммиты и PR с прозрачной проверкой изменений.
